## 轮次 3 · 生成文档 + 搭建骨架

**助手**（基于上述决策，分步落地）

### 配置与构建
- 新建 `package.json` / `vite.config.ts` / `tsconfig.json` / `index.html` / `env.d.ts` / `.gitignore`。

### 项目文档
- [README.md](../../README.md)：介绍、快速开始、架构总览、目录结构。
- [CLAUDE.md](../../CLAUDE.md)：三条核心契约 + 扩展指引。
- [docs/architecture.md](../architecture.md)：分层、DSL 字段表、数据流、扩展配方。

### 代码骨架（四层分离，约 64 个文件）
- **core 层**（框架无关）：[dsl/types.ts](../../src/core/dsl/types.ts)（**SceneDSL = AI 与引擎唯一契约**）、[capabilities/](../../src/core/capabilities/)（灯光/环境/摄像头动画/模型工厂/动画）、[scene/SceneBuilder.ts](../../src/core/scene/SceneBuilder.ts)、[engine/EditorEngine.ts](../../src/core/engine/EditorEngine.ts)、[registry/](../../src/core/registry/)。
- **ai 层**（NL→DSL）：[adapter/](../../src/ai/adapter/)（LLMAdapter 抽象 + MockLLMAdapter + ClaudeAdapter 占位）、[prompt/](../../src/ai/prompt/)、[orchestrator/](../../src/ai/orchestrator/)。
- **domains 层**：[energy/](../../src/domains/energy/)（参考实现，含光伏阵列 builder）、industrial / ict / smart-home（占位）、[_shared/assetBuilder.ts](../../src/domains/_shared/assetBuilder.ts)。
- **three + ui 层**：[three/](../../src/three/)（Renderer/CameraRig/SceneManager/utils）、[ui/](../../src/ui/)（Vue 编辑器界面 + 3 个 Pinia store）。

### 关键架构决策
- `SceneDSL` 是唯一契约：AI 只产 JSON，引擎只消费 JSON。
- 公共能力通过 `EditorContext` 注入，所有领域共享、只组合不重建。
- 领域用 `Domain` 接口注册，新增领域实现接口 + 在 [config/domains.config.ts](../../src/config/domains.config.ts) 加一行。
- AI 走适配器抽象层，先 Mock 跑通全流程。

### 验证
- `npm install`：官方源 `ECONNRESET` 失败 → 切换 npmmirror 镜像成功（58 包，9s）。
- `npx vue-tsc --noEmit`：**退出码 0，零报错**。
- `npm run build`：**成功**（89 模块，1.51s）。

**结果**：骨架可运行。`npm run dev` 后默认 Mock 适配器，切领域 + 输入描述即可看到示例场景（缺真实 GLB 时用占位盒体）。
