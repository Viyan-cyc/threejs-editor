# 2026-06-16 · 多轮场景增删改（不再每轮覆盖）

## 概要

- **问题**：编辑器只支持一轮对话——生成场景后再说「加个物体 / 改一下」会把整个场景覆盖掉。
- **根因**：① AI 层无状态（PromptBuilder 从不告知当前场景已有什么）；② 引擎 `applyScene` 每次先 `clearContent()` 全清重建。
- **方案**：会话状态 + 意图分流（create / edit）+ 增量 ops + 稳定对象 id + 引擎按 id diff。ops 先合并成完整 SceneDSL，引擎再 diff（未改对象复用实例）。
- **改动**：11 个文件（DSL 类型 / normalize / 提示词 / 3 个适配器 / orchestrator / SceneBuilder / SceneManager / EditorEngine）。
- **验证**：`vue-tsc` + `vite build` 通过；增量合并逻辑 13 项冒烟测试全过。

## 轮次 1 · 多轮增删改能力

### 用户

> 这个项目只能进行一轮对话，生成一个场景后，我想要做修改或者新增物体，会把原来的场景覆盖掉。我想要的是能做增删改的场景。除非我说重新生成场景等类似的提示词，才重新生成。

### 助手

定位两层根因后，按「会话状态 + 意图分流 + 增量 ops + 引擎 id diff」方案实现：

- DSL 契约（[types.ts](../../src/core/dsl/types.ts)）：`SceneObjectDSL` 加 `id`；新增 `SceneEditOp`（add/update/remove）、`SceneEditDSL`、`SceneGenerationResult`。引擎仍只消费完整 `SceneDSL`，契约精神不破。
- 合并层（[normalizeDSL.ts](../../src/ai/orchestrator/normalizeDSL.ts)）：`assignIds`（补稳定唯一 id）、`normalizeEditDSL`（容错 op）、`applyEditOps`（ops 合并到上一份完整 DSL）、`isSceneEditDSL`（守卫）。单个坏 op 只记 warning 不中断。
- 提示词（[templates.ts](../../src/ai/prompt/templates.ts) / [PromptBuilder.ts](../../src/ai/prompt/PromptBuilder.ts)）：有 `currentScene` 时切 `EDIT_SYSTEM_PROMPT`，注入当前对象清单（id/type/name/position/子对象数），要求 LLM 输出 `mode:"edit"+ops` 或 `mode:"create"+完整DSL`。
- 适配器（[LLMAdapter](../../src/ai/adapter/LLMAdapter.ts) / [Glm](../../src/ai/adapter/GlmAdapter.ts) / [Mock](../../src/ai/adapter/MockLLMAdapter.ts)）：返回类型放宽为 `SceneDSL | SceneEditDSL`；Mock 按 `opts.currentScene` + 关键词演示 add/remove/重新生成三种行为，离线可跑。
- 引擎（[EditorEngine.ts](../../src/core/engine/EditorEngine.ts)）：新增 `currentScene` / 常驻 `objectRoot` Group / `instanceById` 映射；`applyScene` 区分全量重建与按 id diff；灯光先 clear 再 apply（LightingManager 累加），环境/相机仅在变化时更新（编辑不打断用户轨道视角）。新增 `resetSession()`。
- 辅助：[SceneBuilder](../../src/core/scene/SceneBuilder.ts) 暴露 `buildSingle/buildObjectTree` 并给实例挂 `userData.dslId`；[SceneManager](../../src/three/SceneManager.ts) 加 `clearExcept` 与共享 `disposeAndRemove`。

### 结果

- `npm run typecheck`（vue-tsc --noEmit）通过；`npm run build` 通过（仅有既有的 Three.js 体积告警，与本次无关）。
- 对 `assignIds / applyEditOps` 跑了 13 项冒烟断言：补 id（含子节点）、add 不动原对象、update 部分 transform 合并不丢 params、remove、坏 op 跳过+记 warning、camera 未编辑时不变、edit 覆盖灯光——全部 PASS。
- 行为：首次 / 明确「重新生成」→ 全量重建；其余 → 增量增删改，未提及对象保持原样。
- 范围外（后续）：点击场景树/Inspector 选中对象的可视化增删改；GLM 之外适配器的 edit 适配。

## 验证方式

- Mock 离线：`appConfig.llm.adapter='mock'`，首轮生成示例场景 → 后续发「加一个物体」(add) / 「删掉最后一个」(remove) / 「重新生成」(create 全量重建) 观察不覆盖。
- GLM 真实：发指令后看 Network，第 2 轮起 prompt 应含「当前场景对象清单」；改/增对象时其余不动、相机视角不重置；说「重新生成」才全量重建。
