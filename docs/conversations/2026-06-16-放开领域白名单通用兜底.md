# 对话记录 · 2026-06-16 · 放开领域白名单（通用几何兜底）

> 接 [接入智谱 GLM 模型](./2026-06-15-接入GLM模型.md)。记录约定见 [README.md](./README.md)。

## 概要

- **起因**：能源领域输入"货架 + 黄色集装箱"被领域对象类型白名单拒绝（GLM 守约返回拒绝说明 + 降级默认场景）。
- **根因**：两层限制——① prompt（[BASE_SYSTEM_PROMPT](../../src/ai/prompt/templates.ts) 要求 type 必须来自领域清单）；② 引擎（[SceneBuilder.buildObject](../../src/core/scene/SceneBuilder.ts) 找不到 builder 直接 throw）。
- **决策**：用户选**混合方案**——清单内继续走精致专用 builder，清单外任意 type 走新增的**通用几何 builder**（按 shape/size/color 拼简化 low-poly）。
- 新增 [GenericShapeBuilder.ts](../../src/core/scene/GenericShapeBuilder.ts)（单体+parts 组合+全容错）；SceneBuilder 找不到 builder 改走兜底（不再抛错）；放开 prompt 并补通用参数 schema + 示例。
- **架构调整**：lowPoly 从 `domains/_shared/` 迁到 [three/utils/lowPoly.ts](../../src/three/utils/lowPoly.ts)（避免 core→domains 反向依赖），顺带加 `sphere`。
- [dsl/types.ts](../../src/core/dsl/types.ts) 加 `ShapeKind`/`GenericShapePart`/`GenericShapeParams`。
- `npm run typecheck` 零报错。
- **代价**：清单外对象为简化几何（集装箱=黄色长方体，无波纹/角件），用户已接受。

## 轮次 1 · 放开领域对象类型白名单

**用户**
> 我在能源领域说：我要生成一些货架，上面放置黄色集装箱，返回"[当前领域仅支持"光伏阵列"和"升压站"…无法生成"货架"和"集装箱"。已生成默认的能源场景]"，请告诉我原因，先不用更改代码。
> （随后）能不能不做这种限制？

**助手**

### 诊断（先不改代码）
- 能源领域 [prompts.ts](../../src/domains/energy/prompts.ts) 的 `objectTypes` 只声明 `solar-panel-array`、`power-station`；[PromptBuilder](../../src/ai/prompt/PromptBuilder.ts)+[formatDomainPrompt](../../src/ai/prompt/templates.ts) 把它拼进 system prompt；`BASE_SYSTEM_PROMPT` 下死命令"type 必须来自清单"。GLM 守约拒绝 + 降级。
- "完全不限"在 3D 里绕不开一个现实：引擎必须知道每个物体怎么画，不可能为万物预写 builder → 只能用一个**通用参数化几何 builder** 按 LLM 给的基本形状近似。代价是自由换精致。

### 需求澄清（AskUserQuestion）
- 用户选 **混合方案**：清单内精致 + 清单外通用兜底。

### 实施
1. **lowPoly 迁移**：[domains/_shared/lowPoly.ts](../../src/domains/_shared/) → [three/utils/lowPoly.ts](../../src/three/utils/lowPoly.ts)。理由：GenericShapeBuilder 在 core/scene，引用 lowPoly 若留在 domains 会形成 core→domains 反向依赖；lowPoly 是纯 THREE 工具、归 three 层更正。energy 两个 builder import 路径随之更新。顺带补 `sphere`（低段数多边形球）。
2. **新建 [GenericShapeBuilder.ts](../../src/core/scene/GenericShapeBuilder.ts)**：读 `params`(GenericShapeParams)；有 `parts` 则按组合体（货架=立柱+横梁），否则单 shape；`box/cylinder/sphere/cone` + color。**全容错**：缺 shape→box、缺 size→[1,1,1]、缺 radius/height→0.5/1、color 非法（用 THREE.Color 校验）→灰 0x888888。
3. **[SceneBuilder.ts](../../src/core/scene/SceneBuilder.ts) 兜底**：构造持有 `generic`；buildObject 找不到 builder 时 `buildGeneric(node)`（console.warn 提示）而非 throw。
4. **[templates.ts](../../src/ai/prompt/templates.ts) 放开**：`BASE_SYSTEM_PROMPT` 改为"清单内优先；清单外可用任意 type 但须带通用几何参数（shape/size 或 radius+height/color，复杂体用 parts）"，附黄色集装箱示例。
5. **[types.ts](../../src/core/dsl/types.ts)** 加 `ShapeKind`/`GenericShapePart`/`GenericShapeParams`（`params` 字段仍 `Record<string,unknown>`，builder 内 `as`）。

### 踩坑
- `THREE.ColorRepresentation`（three 0.169）= `number | string | Color`，**不含 `[r,g,b]` 元组**。`resolveColor` 的 RGB 分支最初返回元组导致 TS2322，改为返回 `new THREE.Color(r,g,b)`（Color 是合法 ColorRepresentation）。

### 验证
- `npm run typecheck`：**零报错**。
- 待人工：能源领域"货架+黄色集装箱"→ 简化货架+黄色箱体，不再被拒/报错（控制台有兜底 warn）；光伏阵列回归依旧精致；工业领域"木桌"→ 兜底渲染。

**结果**：领域白名单放开为混合模式，自由生成任意物体（简化几何），精致领域对象不受影响。

**遗留说明**
- 清单外对象永远是简化几何，要精致仍需为该 type 写专用 builder（加入领域 builders）。
- 通用几何的 `parts` 仅一层（不递归），降低 LLM 出错率；极复杂物体表达力有限。
- normalizeDSL 目前未校验通用几何参数（由 GenericShapeBuilder 运行时容错兜住）。
