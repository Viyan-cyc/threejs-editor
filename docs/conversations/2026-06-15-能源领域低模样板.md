## 轮次 1 · 能源领域改用程序化低模

**用户**
> 现在生成的场景都是使用立方体盒子，太简陋了，我想要生成低模

**助手**

### 根因分析
- 所有 builder 走 `ctx.modelFactory.load(key)`，但 [EditorEngine.ts:61-69](../../src/core/engine/EditorEngine.ts#L61-L69) 构造 `AssetRegistry` 后只注册了领域，**从未 `registerAll` 各领域 `assets.ts` 的 `AssetEntry`** → `registry.get(key)` 恒为 `undefined`。
- 即便注册，`public/assets/models/` 也无真实 GLB。
- 结果：`ModelFactory.load()` 恒走"未注册→占位盒体"分支，能源 100% 渲染成半透明蓝盒子。

### 需求澄清（AskUserQuestion）
- 改造范围：**先做 energy 作样板**（而非四领域全做）。
- 低模风格：**写实 PBR 低模**（flatShading + 协调真实配色，非卡通饱和）。

### 实施
1. 新建 [_shared/lowPoly.ts](../../src/domains/_shared/lowPoly.ts)：
   - `lowPolyMaterial(color,{roughness,metalness})` → 固定 `flatShading:true`。
   - `lowPolyMesh(geo,mat)` → 开 `castShadow/receiveShadow`。
   - `box / cylinder / cone` 快捷函数，圆柱圆锥默认 **8 段**（多边形圆是 low-poly 标志）。
2. 改写 [SolarPanelFieldBuilder.ts](../../src/domains/energy/builders/SolarPanelFieldBuilder.ts)：保留 `rows/cols/spacing/tilt` 与 `clone(true)` 阵列逻辑，单块面板改为程序化拼装（深蓝玻璃板面 `0x14233f` + 钢灰边框 + 斜撑支架 + 底座）。
3. 改写 [PowerStationBuilder.ts](../../src/domains/energy/builders/PowerStationBuilder.ts)：混凝土底座 + 双变压器(含瓷绝缘子) + 控制柜 + 避雷针(杆+锥) 的 `THREE.Group` 组合。
4. **不改**：DSL 契约、`mockScenes`、`prompts`、`ModelFactory`/`placeholder`（保留给未来真实资产）。

### 关键决策
- lowPoly 放 `_shared/`（领域共享构建工具），不进 `EditorContext`——纯函数导入即用，不污染场景级能力。
- `clone(true)` 共享几何/材质（N 块面板共用 1 份几何，性能好）；`disposeObject3D` 对共享几何的重复 `dispose` 幂等，安全。
- 整体 `transform` 由 [SceneBuilder.buildObject](../../src/core/scene/SceneBuilder.ts#L37-L54) 调 `applyTransform` 统一施加，builder 内不处理。

### 验证
- `npm run typecheck`（vue-tsc --noEmit）：**零报错**。
- 待人工：`npm run dev` → 选能源领域 → 发任意消息，视口应见 6×10 倾斜深蓝光伏阵列 + 组合式升压站，而非蓝盒子。

**结果**：能源领域样板完成，写实 PBR 低模可立即生效。其余三领域仍为占位盒体，待后续铺开。

**遗留说明**
- **资产未注册 bug**（EditorEngine 未 `registerAll` 资产）：本次不影响（能源 builder 不再依赖资产），接真实 GLB 时需一并修，届时 builder 可改为"资产优先、lowPoly 回退"。
- 铺开 industrial / ict / smart-home 时复用同一套 lowPoly 工具与 builder 改写模式。
